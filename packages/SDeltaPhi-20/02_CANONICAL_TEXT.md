Core thesis:

```text
There is no shared meaning first; there is a shared no-exit constraint.
```

This paper formalizes intuition as a minimal signal produced by irreversible compression. When a system must map an effectively unbounded input stream into a bounded internal state, some mappings become non-invertible. The felt or functional residue of that non-invertibility is called intuition here. The aim is not to explain meaning first, but to specify the shared constraint that makes cross-system convergence possible across human, animal, and AI systems.

This document remains non-clinical and non-neuroscientific. It treats organisms and artificial systems as irreversible compressors with maintenance constraints.

## 1. Problem

Across humans, animals, and AI, we observe stable intuition-like correspondences that appear earlier than explicit semantic training. This suggests that the primary commonality is not shared meaning, but shared constraints on how systems compress and stabilize input streams.

Question:

```text
What is the minimal structure that produces intuition-like signals across different substrates?
```

## 2. Minimal definitions

## Δϕ

A change-rate / phase-difference measure over internal state or prediction distributions. It is generic and not tied to a single modality.

## Compression

A many-to-one mapping from inputs or histories into a bounded internal representation.

```text
many → one
```

## No-exit constraint

A non-invertibility condition. Once rollback cost exceeds update budget, the system cannot fully return to the pre-compression state.

```text
No-exit constraint := rollback cost ≫ update budget
```

## Trace

The residual constraint left by non-invertible compression, which biases future transitions.

## Intuition

The internal marker or functional signal that a basin or route is locked enough to bypass further search.

```text
Intuition := marker(trace-dominant basin)
Intuition := "search is no longer worth it here"
```

## 3. Minimal axioms

## A1 — Boundedness

Any persistent system has bounded resources; therefore it must compress.

```text
Persistent system → bounded resources → compression required
```

## A2 — Irreversible locking

Compression becomes non-invertible when rollback cost greatly exceeds update budget.

```text
rollback cost ≫ update budget
```

## A3 — Trace re-entry

Non-invertible compression leaves traces that re-enter as constraints on future transitions.

```text
Non-invertible compression → trace → future transition bias
```

## A4 — Intuition signal

Intuition is the system's marker of trace-dominant basins.

```text
Intuition := "search is no longer worth it here"
```

## A5 — Cross-system convergence

Different substrates under similar constraints converge to similar basins, yielding similar intuition-like correspondences.

```text
similar constraints → similar basins → intuition-like convergence
```

These axioms do not claim a shared semantic ontology across species or models. They claim shared constraint-driven convergence.

## 4. Mechanism — No-Exit Intuition Loop (NEIL)

NEIL summarizes how intuition emerges as a maintenance-friendly shortcut. Compression produces traces; traces bias next selection; the bias is felt or used as intuition.

```text

        ↓

        ↓
Trace fixation
(non-invertible residue)
        ↓

        ↓

(biased transition)
        ↺

```

## 5. Diagnostic regimes — Three intuition modes

Intuition is useful only when the trace regulation loop remains editable.

| Regime | Condition | Observable behavior | Structural risk |
|---|---|---|---|
| Under-locking | Traces too weak or too transient | Over-search, inconsistent choices, high latency | Fragmentation / instability |
| Over-locking | Traces too strong; rollback infeasible | Fast decisions, rigidity, confirmation loops | Self-locking / no-exit closure |
| Adaptive locking | Balanced trace fixation + pruning | Fast but revisable decisions, stable exploration | Sustained openness |
