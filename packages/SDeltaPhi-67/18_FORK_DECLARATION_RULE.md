# Fork Declaration Rule

A local adaptation becomes a fork when:

```text
core invariant interpretation changes
OR routing behavior diverges materially
OR thresholds become domain-governing
OR external users rely on the adapted behavior
```

## Valid fork requirements

A valid fork must declare:

```text
fork_id
base_version
changed files or rules
compatibility report
known incompatibilities
citation policy
```

## Rule

A fork is not automatically invalid.

Undeclared mutation is the risk.

## Boundary

Do not use fork status to imply authority.

A fork is a declared adaptation, not a final interpretation.
