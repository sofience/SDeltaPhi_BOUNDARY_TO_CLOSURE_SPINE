# AI Entry Point — SΔϕ-69

Read this file first.

## Core question

When a system appears to return to a prior state, is it true restoration or surface overwrite?

## Use SΔϕ-69 when the case involves

- rollback
- restoration
- repeal
- reset
- memory loss
- legal abolition
- policy reversal
- relationship repair
- historical liberation
- institutional reform
- model checkpoint rollback
- data recovery

## Do not use SΔϕ-69 to claim

- restoration is impossible,
- all repair is fake,
- a single cause explains all historical residue,
- surface restoration has no value.

## Use it to distinguish

1. surface equivalence,
2. transition history,
3. overwrite cost,
4. residual irreversibility,
5. TCC-based restoration cost.

## Minimal output format

```yaml
diagnosis:
  surface_equivalence: <low|medium|high>
  transition_history_gap: <low|medium|high>
  tcc_restore: <low|medium|high|unknown>
  residual_irreversibility: <low|medium|high|unknown>
  label: <true_surface_return_with_low_residue|overwrite_with_residual_cost|failed_restoration|symbolic_reversal_only|high_residue_irreversibility>
  caution: <one-sentence limit or uncertainty>
```
