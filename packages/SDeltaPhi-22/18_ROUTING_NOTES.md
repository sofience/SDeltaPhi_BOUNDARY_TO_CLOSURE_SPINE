SΔϕ-22 should be called when the task involves authority editing, re-entry governance, triggers, rollback handles, finite-cost editability, authority debt, drift, closure, or open persistence.

```text

```

## Closure / openness route


## Governance route

```text
SΔϕ-18 → SΔϕ-21 → SΔϕ-22
```

## Call SΔϕ-22 when the task involves

- authority editing;
- authority revision triggers;
- finite-cost editability;
- re-entry governance;
- rollback handles;
- trigger handling;
- conflict as editing material;
- authority debt control;
- open persistence;
- drift / under-authority;
- closure / over-authority;
- co-environmentalization.

## Do not call SΔϕ-22 merely because the task mentions authority

If the task asks for legal, moral, or political authority theory, use domain-specific analysis.

If the task asks about judgment, route first through SΔϕ-21.

If the task asks about intuition, route through SΔϕ-20.

If the task asks about coupled environments, route downstream to a relational dynamics module when available.
