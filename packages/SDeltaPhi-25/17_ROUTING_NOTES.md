# Routing Notes

SΔϕ-25 should be called when the task involves pre-detection, latent difference, threshold crossing, event registration, structural invisibility, detection bandwidth, or innovation before recognition.

## Primary route

```text
SΔϕ-03 → SΔϕ-21 → SΔϕ-22 → SΔϕ-24 → SΔϕ-25
```

## Innovation route

```text
SΔϕ-20 → SΔϕ-21 → SΔϕ-24 → SΔϕ-25
```

## Call SΔϕ-25 when the task involves

- pre-detection;
- latent difference;
- operative but undetected difference;
- threshold crossing;
- event registration;
- structural invisibility;
- detection bandwidth;
- false absence judgment;
- innovation before recognition;
- authority shaping visibility;
- registered event vs latent process.

## Do not call SΔϕ-25 merely because something is unknown

If the task asks about empirical hidden causes, use domain evidence.

If the task asks about innovation, route also through SΔϕ-24.

If the task asks about authority and detectability, route also through SΔϕ-21 and SΔϕ-22.

If the task asks about irreversibility, route also through SΔϕ-03.
