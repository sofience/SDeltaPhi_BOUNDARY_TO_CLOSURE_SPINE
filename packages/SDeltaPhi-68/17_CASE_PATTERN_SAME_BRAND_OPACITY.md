# Case Pattern B — Same-Brand Capability Opacity

## Case type

Same-brand model-family opacity.

## Pattern

A user asks an accessible model about another model, tier, or capability in the same brand family.

The accessible model cannot reliably identify the referenced capability, product tier, or gated route.

## Surface interpretation

```text
The user is using the brand,
so the model should know the brand's full capability map.
```

## SΔϕ-68 interpretation

```text
Access to the brand name
≠
access to the brand's full intelligence map.
```

## Cost Theater

The burden shifts to the user:

```text
identify actual model tier
verify product-family claims
discover gated capability routes
separate brand access from capability access
```

## Boundary

The careful preservation of uncertainty is not itself a failure.

The Cost Theater lies in the hidden route-discovery burden.
