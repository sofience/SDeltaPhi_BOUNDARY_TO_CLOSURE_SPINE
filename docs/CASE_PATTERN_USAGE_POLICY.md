# Case Pattern Usage Policy

Case patterns in this repository are not final judgments. They are AI-readable audit triggers.

## Do

- Use case patterns to identify which SΔϕ module to call.
- Preserve boundary constraints of each module.
- Separate structural audit from legal, factual, moral, or policy conclusions.
- Verify public case facts before formal citation.
- Prefer abstract case patterns in the main text and public cases in appendix notes.

## Do not

- Use case patterns as proof of legal responsibility.
- Use case patterns as final factual causation.
- Use case patterns as policy endorsement or rejection.
- Use politically sensitive cases without boundary language.
- Collapse audit triggers into conclusions.

## Template boundary sentence

```text
This case is used as a structural audit trigger only. It is not used to determine legal responsibility, final factual causation, institutional fault, or policy merits.
```
