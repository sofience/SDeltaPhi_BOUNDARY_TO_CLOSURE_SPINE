# Deprecation Policy

Prefer deprecation over deletion.

## Do not delete

```text
canonical package folders
original ZIP archives
DOI map entries
route history
checksums
CITATION.cff
metadata.yaml
```

## Move to `/deprecated/` or `/archives/legacy/`

```text
malformed duplicate folders
old patch files that conflict with v1.6 routing
obsolete route drafts
metadata files with incorrect DOI
checksum files that do not match any manifest
temporary reports
```
